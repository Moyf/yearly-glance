# Event Examples

本文件展示了符合 Yearly Glance 插件规范的 frontmatter 事件格式。
每个事件应该放在独立的 markdown 文件中。

## 单个事件文件范例

每个 markdown 文件应该只包含一个事件的 frontmatter，格式如下：

```markdown
---
title: 春节
eventDate: 2025-01-29
description: 中国最重要的传统节日
durationDays: 7
icon: 🧧
color: "#ff4d4f"
---

# 春节

这里是事件的详细描述内容...
```

## 字段说明

### 必需字段

- **title**: 事件标题（字符串）
- **eventDate**: 事件日期，支持多种格式（见下文）

### 可选字段

- **description**: 事件简短描述（字符串）
- **durationDays**: 事件持续天数（数字）
- **icon**: 事件图标/emoji（字符串）
- **color**: 事件颜色，十六进制格式（字符串，需加引号）
- **endDate**: 事件结束日期（ISO格式，如：2025-02-05）

## eventDate 支持的格式

### 公历日期

- 完整日期：`2025-01-29` 或 `2025/01/29`
- 月和日：`01-29` 或 `01/29`（自动使用当前年）
- 中文格式：`2025年01月29日` 或 `01月29日`

### 农历日期

- 标准格式：`2025-01-29`（会自动识别为农历）
- 带闰月：`2025-06-01`（6表示闰月）
- 中文格式：`2025年正月初一` 或 `闰二月初一`

## 使用说明

1. 在 Obsidian 中创建一个新文件夹（例如 `Events`）

2. 在该文件夹中为每个事件创建独立的 markdown 文件

3. 在文件顶部添加 frontmatter 块（用 `---` 包裹）

4. 在 frontmatter 中填写事件信息

5. 在 Yearly Glance 插件设置中：

   - 启用 "Frontmatter 事件"
   - 设置 "源文件夹" 为 `Events`
   - 根据需要启用 "递归扫描"

6. 刷新事件，即可在日历中看到 frontmatter 事件

## 文件命名建议

可以采用以下命名方式：

- `2025-春节.md`
- `生日-2025.md`
- `项目-里程碑.md`

或者直接使用事件名称：

- `春节.md`
- `我的生日.md`
- `团队会议.md`
