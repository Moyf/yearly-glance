# v4.0 新特性

本文档介绍 Yearly Glance v4.0 的重点新增特性。

---

## 日记事件（Daily Note Events）

现在可以直接从日记笔记中创建事件。事件存储在日记 frontmatter 的列表属性中，自动显示在年历上。

### 使用方法

1. 在 设置 → 日记事件 中**启用**
2. **选择来源**：核心「日记」插件或「Periodic Notes」插件
3. **设置属性名**：存储事件的 frontmatter 属性（默认：`events`）

### 日记示例

```yaml
---
events:
  - "🧩 开发插件"
  - "📚 阅读"
  - "团队会议"
---
```

每一项都会成为当天的一个独立事件。文本开头的 emoji 会被自动提取为事件图标。

### 创建日记事件

- 点击 ✏️ 按钮 → 选择「日记事件」标签 → 输入事件名和日期
- 事件会写入对应日记的 frontmatter 属性
- 如果日记文件不存在，会自动创建

### 日期迁移

编辑日记事件并修改日期时：
- 事件会从**原来的日记**中移除
- 并**添加到新的日记**（如不存在会自动创建）
- 表单中会显示提示：「事件将从原来的日记（2026-04-22）中转移到新的日记」

## 笔记事件（Note Events）

使用带有 `event_date` frontmatter 属性的笔记作为事件来源。这让你可以在事件笔记中附加丰富内容 — 描述、链接、清单 — 超越简单日历条目的限制。

### 使用方法

1. 在 设置 → 笔记事件 → 默认笔记事件路径 中设置根文件夹（如 `Events/`）
2. 在该文件夹中创建带有所需 frontmatter 的笔记
3. 笔记自动作为事件显示在年历上

### 笔记示例

```yaml
---
title: 团队外出
event_date: 2026-06-15
duration: 3
icon: 🏢
color: "#3498db"
description: 年度团队建设活动
---

## 日程
- 上午：战略研讨
- 下午：团队活动
```

### 可用属性

| 属性 | 默认属性名 | 是否必需 | 说明 |
|------|-----------|---------|------|
| 标题 | `title` | 否（使用文件名） | 事件显示名称 |
| 日期 | `event_date` | **是** | 事件日期（YYYY-MM-DD） |
| 持续天数 | `duration` | 否（默认 1） | 事件持续天数 |
| 图标 | `icon` | 否 | 自定义 emoji |
| 颜色 | `color` | 否 | 十六进制颜色代码 |
| 描述 | `description` | 否 | 附加说明 |

所有属性名可在 设置 → 笔记事件 中自定义。

### 从年历创建笔记事件

点击 ✏️ 按钮 → 选择「笔记事件」标签 → 填写事件信息。系统会在配置的根文件夹中自动创建笔记文件并写入对应的 frontmatter。

### Bases 视图集成

如需高级筛选，可使用 Obsidian 的 **Bases** 功能：

1. 创建 `.base` 文件，用查询选取事件笔记
2. 添加 **Yearly Glance** 视图
3. 在视图选项中映射属性名

这样可以创建多个日历视图，使用不同的事件来源和筛选条件。

> 关于 Bases 的更多信息，参见 [Obsidian Bases 文档](https://help.obsidian.md/bases)。

## 笔记事件文件名格式

笔记事件（Bases 事件）现在支持配置文件名格式，可以按日期组织到不同的文件夹层级。

### 设置

**设置 → 笔记事件 → 文件名格式**

### 语法

| 通配符 | 含义 | 示例 |
|--------|------|------|
| `{event_name}` | 事件标题 | `我的事件` |
| `YYYY` | 4 位年份 | `2026` |
| `MM` | 2 位月份 | `04` |
| `DD` | 2 位日期 | `29` |
| `YY` | 2 位年份 | `26` |
| `[文本]` | 字面字符（不解析为日期） | `[YG]` → `YG` |

### 示例

| 格式 | 事件名 "会议"，日期 2026-04-29 | 最终路径 |
|------|-------------------------------|----------|
| `{event_name}`（默认） | — | `Events/会议.md` |
| `YYYY-MM/YYMMDD_{event_name}` | — | `Events/2026-04/260429_会议.md` |
| `YYYY/MM/{event_name}` | — | `Events/2026/04/会议.md` |
| `[Event]_{event_name}_MMDD` | — | `Events/Event_会议_0429.md` |

中间目录会自动创建。设置页面会根据当前日期实时预览最终路径。

## 右键菜单

右键点击年历上的任意事件，会弹出原生菜单：

| 操作 | 显示条件 | 功能 |
|------|---------|------|
| **编辑** | 所有事件 | 直接打开编辑表单（跳过预览） |
| **打开笔记** | 笔记/日记事件且有源文件 | 在新标签页打开源笔记 |
| **删除** | 所有事件 | 弹出确认对话框后删除 |

删除行为因事件类型而异：
- **配置事件**（节日/生日/自定义）：从插件数据中移除
- **笔记事件**：源笔记文件移入回收站
- **日记事件**：从日记 frontmatter 属性列表中移除该值

## 编辑窗口删除按钮

事件编辑表单左下角新增了**删除按钮**，无需切换到事件管理器即可快速删除事件。

- 点击后弹出确认对话框
- 支持所有事件类型的对应删除操作

---

## 记住上次事件类型

创建新事件时，表单会自动默认为上次选择的事件类型。例如上次创建了日记事件，下次打开表单时会自动切换到日记事件标签页。

此偏好在会话间持久保存。

## UI/UX 改进

### 颜色选择器
- 预设颜色下拉选项前显示**圆形色块**
- 响应式布局：窄宽度时自动换行
- 下拉框靠右对齐，最大宽度 50%

### 响应式标签页
- **事件管理器标签**：正常宽度显示 emoji + 文字，容器窄（≤600px）时仅显示 emoji
- **事件表单标签**：默认只显示文字，视口窄（≤600px）时仅显示 emoji

### 事件预览弹窗
- 来源文件路径简化为仅文件名（hover 显示完整路径）
- 弹窗最大宽度限制为 360px
- 所有操作按钮统一使用 Tooltip 组件

### 年历标题
- 年份数字：衬线字体，1.5em，下划线装饰
- hover 时年份数字和下划线变为强调色
- 标题和年份数字 baseline 对齐

### 事件图例
- 容器宽度 ≤600px 时折叠为仅 emoji 模式

### 事件管理器
- 头部操作按钮空间不足时换行显示
- 事件详情仅显示有意义的属性值
- 添加按钮改为与相邻按钮一致的圆形 ✏️ 图标

## 日期校验增强

非法日期输入现在会被正确拦截：
- `2026-04-220` → 错误：日期范围无效
- `2026-13-01` → 错误：月份超出范围
- `2026-02-30` → 错误：日期超出当月最大天数

日期输入框实时显示 ✓ 或 ⚠ 校验指示。
